# Research

A living record of what we've found, what we're learning, and what we're testing.

This page is intentionally empty in the starter stack — populate it with the work your org does. A few patterns that work well here:

- Campaign briefs and policy summaries
- Meeting notes from working groups
- Annotated reading lists
- Members-only deep-dives (just don't link to them from the public nav)

To add a new sub-page: create a file like `docs/some-topic.md`, then add an entry under `Research` in `mkdocs.yml`'s `nav:` section.
